# 📬 Pull Request 규칙

Pull Request는 단순 병합 요청이 아니라 **팀원 간의 커뮤니케이션 도구**입니다.

---

## 🏷 PR 제목 규칙

커밋 컨벤션과 동일한 형식을 사용합니다.

```text
feat: 랜딩 페이지 구현
fix: API 응답 에러 처리
```

---

## 🧾 PR 본문 작성 가이드

PR 생성 시 아래 항목을 작성해주세요.

```md
## 🔑 Key Changes

- 변경된 주요 작업 내용을 요약합니다.

### 📸 Screenshots (Optional)

- UI 변경 사항이 있을 경우 스크린샷 또는 GIF 첨부

## 🙋‍♂️ To Reviewers

- 리뷰 시 중점적으로 봐주었으면 하는 부분
- 논의가 필요한 사항

## 🔗 Issues

- 관련 GitHub Issue 번호 (예: #12)
```

---

## ✅ PR 체크리스트

- [ ] 브랜치 전략을 따랐습니다.
- [ ] 커밋 메시지 컨벤션을 준수했습니다.
- [ ] 불필요한 콘솔 로그를 제거했습니다.
